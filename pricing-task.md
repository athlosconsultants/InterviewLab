# Pricing Page Improvements

## Context

These improvements target the pricing page experience across:

- Desktop browsers
- Mobile web browsers (Safari, Chrome, etc.)
- In-app browsers (TikTok, Instagram, Facebook, etc.)

All button style updates reference the "See What Your Answers Are Missing" button found in the "See What You're Missing" section on the mobile landing page as the design source of truth.

---

## Task 1: Update "Begin Training" Button Style

**Location**: Plan type card(s) on the pricing page

**Goal**: The "Begin Training" button should match the visual style and color of the "See What Your Answers Are Missing" button from the mobile landing page's "See What You're Missing" section.

**Requirements**:

- Match the exact button style (border, padding, border-radius, typography, font-weight, etc.)
- Match the exact color scheme (background color, text color, hover states)
- Ensure consistency across desktop, mobile web, and in-app browsers
- Maintain any interactive states (hover, active, focus)

**Scope**:

- Locate the "Begin Training" button on the plan type card(s)
- Locate the "See What Your Answers Are Missing" button on the mobile landing page for style reference
- Apply matching styles to ensure visual consistency
- Test across all supported platforms and browsers

## Task 4: Reposition "Just try it already" Card

**Current Location**: Somewhere within the main pricing page content

**New Location**: Below the "Common Questions" section, just above the footer

**Goal**: Move the "Just try it already" card to the bottom of the page to serve as a final call-to-action before the footer.

**Requirements**:

- Locate the "Just try it already" card
- Locate the "Common Questions" section
- Locate the page footer
- Reposition the card to appear after "Common Questions" and before the footer
- Ensure proper spacing above and below the card in its new position
- Maintain card styling and responsiveness in the new location
- Verify layout integrity on both desktop and mobile viewports

**Design Considerations**:

- The card should feel like a natural final CTA at the bottom of the page
- Spacing should be consistent with other section gaps on the page
- The card should remain fully visible and accessible in its new position

**Scope**:

- Adjust component order/positioning in the page layout
- Verify spacing and alignment in the new position
- Test across desktop, mobile web, and in-app browsers
- Ensure no layout breaks or overflow issues

## Button Style Consistency Note

- Brand consistency
- Visual hierarchy clarity
- Professional, cohesive user experience

1. Visual correctness on desktop browsers (Chrome, Safari, Firefox)
